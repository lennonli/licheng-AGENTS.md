# licheng-AGENTS.md

本仓库用于保存法律工作相关的 Codex/Agent 指令文件，便于在不同项目中复用统一的工作立场、法律核验边界、文档格式和合同审查起草流程。

## 文件说明

- `法律业务通用指令.md`：法律业务通用指令。覆盖默认服务立场、保密要求、法律依据核验、反编造规则、文档修订模式、起草默认值、Word 文档格式、律所署名、跨境事项提示和交付方式等通用要求。
- `合同审查和起草.md`：合同审查与法律文件起草专项指令。重点规定合同审查流程、风险分级、缺失条款补充、文字和格式校对、修订模式、合同起草要点、编号规则、签署页安排和版本命名规则。
- `法律PPT设计.md`：法律及专业服务 PPT 设计风格指令。重点规定简洁专业的视觉风格、页面结构、字体层级、信息呈现、视觉元素使用、既有 PPT 修改边界和新建 PPT 初稿确认流程。
- `法律尽调报告提示词.md`：法律尽职调查报告专项提示词。当前先收录股权历史沿革部分，覆盖工商内档核查、股权变动底稿 PDF 拆分、历史沿革正文起草、补充材料清单和交付验收要求，后续可继续追加其他尽调章节。

## 使用建议

- 需要通用法律工作规则时，优先使用 `法律业务通用指令.md`。
- 任务集中在合同审查、协议修订或合同起草时，可在 `法律业务通用指令.md` 的基础上叠加使用 `合同审查和起草.md`。
- 任务涉及法律培训、项目汇报、会议展示或专业服务 PPT 制作与修改时，可叠加使用 `法律PPT设计.md`。
- 任务涉及法律尽职调查报告起草、工商内档核查、股权历史沿革梳理或尽调底稿整理时，可叠加使用 `法律尽调报告提示词.md`；当前版本仅覆盖股权历史沿革部分。
- 如具体项目已有客户身份、交易背景、特殊争议解决安排或文件模板，应以具体项目指令优先，并与本仓库规则保持一致。

## 维护规则

- 每次更新本仓库内任何 Markdown 文件时，应同步更新本 README 的“更新记录”部分。
- 更新记录应写明更新日期、涉及文件和具体优化内容，便于后续追踪指令演进。
- 更新记录按“同一日期 + 同一文件”汇总填写；同一天内同一个文件即使多次更新，也只写一行，并在该行内概括当天对该文件的全部重要优化内容。
