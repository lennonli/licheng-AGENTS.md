# 法律尽调报告提示词
本文件用于沉淀法律尽职调查报告的专项提示词，当前版本覆盖报告通用撰写规范及9项核心章节/附件的撰写要求；后续可按统一格式继续追加重大合同、税务、环保、劳动用工、诉讼仲裁等其他章节内容。

---

## 一、通用撰写规范
### （一）整体任务目标
以现有法律尽职调查报告模板为基础，结合目标公司企查查企业信用报告（专业版）、工商内档、专项尽调材料及用户确认信息，分章节、分版本逐步撰写/更新目标公司法律尽职调查报告；每完成一个阶段生成一个版本（V1、V2、V3……），经用户确认后进入下一阶段。

### （二）文件命名规范
**命名格式：[目标公司简称]-[文件类型]-ABL-[日期YYYYMMDD]-V[版本号]**
- 示例：`XX科技-法律尽职调查报告-ABL-20260716-V1.docx`
- 版本号依次递增：V1、V2、V3……
- 文件名版本号后可附简短中文修改说明，用于区分每版核心改动。

### （三）整体工作流程
#### 第一步：修改释义与基础信息（V1）
1. **提取主体信息**：读取目标公司企查查报告，整理需替换的主体信息：
   - 公司全称及简称（简称一般取“XX科技”形式）、曾用名/前身
   - 机构股东（自然人股东无需释义）、子公司及对外投资企业、实际控制人
2. **释义表替换**：
   - 以模板释义表为基础，替换原模板中的公司主体、股东、关联方为目标公司对应主体
   - 仅列入机构/企业类主体，自然人姓名不纳入释义
   - 根据目标公司所属行业及交易类型调整法律法规释义：删除无关行业特定法规，保留《民法典》《公司法》《证券法》等通用法规
   - 同步更新报告全称、报告期间
3. **引言与封面同步修改**：替换封面主标题、引言中的公司全称，统一更新公司简称引用。

#### 第二步：确认委托方及交易背景
1. **明确委托方身份**：
   - IPO尽调：委托方为标的公司本身，引言致送方写标的公司全称
   - 投资尽调：委托方为投资方，需用户提供投资方全称及简称
2. **调整引言与使用目的**：
   - 致送方统一为“致：[委托方全称]”，委托段表述随双方关系调整
   - IPO尽调：使用目的改为“本报告仅供公司首次公开发行人民币普通股股票并上市之目的使用”
   - 投资尽调：使用目的调整为“本报告仅供[投资方]拟投资[标的公司]之目的使用”
   - 保密条款保留不变。

#### 第三步及以后：逐章替换正文内容（V2及以后）
按报告章节顺序逐章替换模板正文，依据企查查报告、工商内档及公司提供的尽调材料填充内容；每完成1个或多个章节更新，生成对应新版本。

### （四）通用操作要求
1. 始终以官方模板为基础，坚持最小必要修改原则，保留模板原有结构、编号、格式与行文风格。
2. 采用docx解包（unpack）→ 编辑XML → 重新打包（pack）的方式处理文件，确保格式不丢失。
3. 所有版本均基于用户最新修改确认的版本续改，保留用户删改内容与表述口径；不使用修订模式，生成清洁版并按版本号另存。
4. 每个版本生成后直接打开供审查，同时列明本次修改内容，经用户确认后再进入下一步。
5. 现有资料无法确认的内容统一用`【】`占位，不得自行推断、编造事实；对无法核实的事项标注`【待核验】`。
6. 遇到不确定事项（如委托方身份、报告期间、事实细节等），主动向用户确认。
7. 章节修改严格限定范围：仅处理指定章节/附件，不改动报告其他内容；手工编号与自动编号不得重复。

### （五）需用户提供/确认的基础信息
- 目标公司全称及对应企查查报告
- 委托方身份（标的公司本身 / 投资方）；若为投资尽调，需提供投资方全称及简称
- 报告期间（如2024年1月1日至2026年6月30日）
- 目标公司所属行业（用于调整法律法规释义）
- 对应章节的专项材料（工商内档、知识产权清单、三会文件等）

## 二、分章节专项撰写要求
按尽调报告常规章节顺序排列，各章节专项要求叠加通用规范执行。

### （一）第一章 公司的基本情况
1. 参照模板原有内容与表格，根据公司最新工商、信用资料进行完整替换。
2. 无法确认的内容用`【】`占位，不新增模板外的冗余内容。

### （二）第二章 股权历史沿革
#### 1. 使用规则
- 本章节提示词与通用规范叠加使用；如具体项目已有客户身份、交易背景、报告模板或特别要求，以具体项目指令优先。
- 使用前需将目标公司名称、工商内档PDF路径、参考报告路径、底稿与报告输出目录等占位内容替换为真实信息。
- 仅处理股权变动相关事项，不扩展经营范围、注册地址、法定代表人、董监高备案、一般章程备案等非股权事项；如非股权信息仅为解释股权变动所必需，应简洁陈述并明确关联性。

#### 2. 任务目标
根据工商内档、已有底稿及参考尽调报告的表述方式，完成以下工作：
1. 梳理目标公司自设立以来全部股权变动相关的历史沿革。
2. 仅提取分析股东、股权比例、认缴/实缴出资额、注册资本、股权转让、增资、减资、股改、代持还原、员工持股平台等影响股权结构的事项。
3. 将每次股权变动对应的工商底稿从原始PDF中拆分，整理为单独PDF。
4. 参照参考报告的表达方式起草尽调报告“历史沿革”章节，仅写该部分内容。
5. 针对缺失材料、证据链断点形成详细补充材料清单。

#### 3. 输入材料
- 目标公司名称
- 工商内档PDF（绝对路径）
- 参考尽调报告DOCX（绝对路径）
- 已有历史沿革梳理表/底稿目录（无则填“无”）
- 底稿PDF输出目录、历史沿革初稿输出目录

#### 4. 核查与梳理要求
1. 以工商内档为核心事实来源，重点核对设立登记、变更登记申请书、股东会/董事会决议、章程/章程修正案、股权转让协议、增资协议、验资报告等原始文件。
2. 扫描件PDF需先OCR识别，并人工复核每次股权变动的关键事实：变更日期、决议日期、转让方/受让方、转让比例、转让出资额、转让价款等。
3. 公司设立作为历史沿革起点单独梳理，重点提取设立时间、注册资本、初始股东、认缴出资额、持股比例、出资期限、公司章程及设立依据。
4. 历次股权变动按时间顺序编号，不得遗漏；工商变更日期、股东会决议日期、协议签署日期、营业执照核发日期不一致时，分别说明并核查原因。
5. 每次股权变动形成结构化记录，至少包含：序号、变更时间、变更事项、变更依据文件、变更前/后股权结构、对应工商内档页码、对应拆分底稿PDF文件名、待核验事项。
6. 发现股权转让价款、税款缴纳、出资实缴、优先购买权放弃、配偶同意、国资/外资审批、持股平台内部份额、代持安排等材料缺失，逐一记录并纳入补充清单。

#### 5. 底稿PDF整理要求
1. 仅拆分历史沿革相关底稿，不纳入经营范围、地址、法定代表人、董监高等无关变更材料。
2. 每次股权变动（含公司设立）制作一个独立PDF，尽量包含该次变动的完整证据链（变更申请书、决议、协议、章程/修正案、验资报告等）。
3. PDF命名规则：`序号-日期-变更事项.pdf`；日期优先使用`YYYYMM`，需精确到日时使用`YYYYMMDD`。
4. 命名示例：`1-201703-第一次股权变动（A转让10%股权给B）.pdf`；多方转让可概括为`A等转让合计30%股权给B等`。
5. 拆分后复核PDF页数、页面方向、可读性、是否漏页/误纳入无关材料，并同步更新股权历史沿革梳理表。

#### 6. 报告正文写作要求
1. 参照参考报告历史沿革章节的层级、语气与表达方式，不得复制参考报告事实内容。
2. 建议章节结构：
   - 二、公司的历史沿革
   - （一）公司设立及历次股权变动
     - 1、公司设立
     - 2、第一次股权变动
     - 3、第二次股权变动……
   - （二）历史沿革相关核查事项
3. 正文开头统一表述：`根据公司提供的工商内档资料、公司章程、股东会决议、股权转让协议及本所律师核查，公司自设立以来的股权历史沿革如下……`
4. 每次股权变动按“事实依据 + 变更内容 + 变更后股权结构”顺序撰写：
   - 事实依据：列明股东会决议、协议、章程修正案、验资报告、工商登记/营业执照等文件
   - 变更内容：说明转让方/受让方、转让比例、对应出资额、增/减资金额、注册资本变化
   - 股权结构：以表格列示变更后的股东、认缴出资额及持股比例
5. 股权结构表建议字段：序号、股东姓名/名称、认缴出资额（万元）、实缴出资额（万元，可核实则填）、持股比例、出资方式、出资期限、备注。
6. 1,000及以上数字使用千位分隔符；金额、比例、日期必须与底稿一致。
7. 底稿未显示价款支付、税费缴纳、出资实缴、优先购买权放弃等情况的，在“历史沿革相关核查事项”中列明，正文不作推断。
8. 存在代持、回购、股权激励、持股平台、实控人变化、增资估值异常、出资瑕疵、减资债权人通知、国资/外资审批等特殊情况，单独设段说明并列示待核查事项。
9. 文字客观克制、可追溯，避免使用“合法有效”“不存在争议”等结论性表述，除非证据链充分且完成外部核验。

#### 7. 补充材料清单（按需细化）
1. **工商登记及章程资料**：完整工商内档、历次变更登记申请书/通知书/营业执照、历次公司章程/修正案、股东名册、出资情况表、最新章程与营业执照、信用公示系统报告、缺页/模糊部分的清晰扫描件。
2. **出资实缴及注册资本资料**：历次出资银行进账单/回单、验资报告/附表/询证函、会计凭证与账簿、审计报告、非货币出资评估报告/权属证明/交割文件、出资瑕疵说明及整改证明、增减资相关决议与债权人通知/公告/清偿说明。
3. **股权转让交易资料**：历次股权转让协议及补充协议、价款支付凭证/流水/确认书、定价依据/评估/审计报告、低价/零对价转让原因说明、股权交割与工商变更确认文件。
4. **税费缴纳资料**：个税/企业所得税申报表及完税证明、印花税缴纳凭证、税务机关涉税证明、未缴税原因说明及补缴安排。
5. **股东内部决策及权利放弃资料**：各方内部决策文件、其他股东优先购买权放弃函、自然人股东配偶同意函、国资/外资/上市公司等特殊主体审批备案文件、授权委托书及主体资格证明。
6. **股权权属及限制资料**：股权权属清晰确认函、无代持/质押/冻结/争议/回购/对赌确认函、出质/冻结/诉讼查询结果、代持解除与纠纷解决文件。
7. **员工持股平台及激励资料**：合伙协议、合伙人名册、出资证明、份额转让文件、激励计划与授予/退出协议、价款支付与税务资料、平台控制关系说明、离职份额处理文件。
8. **实际控制人及特殊事项资料**：实控人认定及变化说明、一致行动/表决权委托协议、红筹拆除/VIE解除/股改/瑕疵整改资料、股东失联/注销/继承/合并分立证明、历史沿革真实性书面说明。

#### 8. 输出成果与验收要求
最终交付4项成果：
1. 股权历史沿革梳理表（含序号、日期、变更事项、变更依据、变更后股权结构、原PDF页码、对应底稿PDF、待核验事项）。
2. 历次股权变动底稿PDF（按命名规则，每次变动一个文件）。
3. 尽调报告历史沿革部分初稿（Word格式，仅含该章节）。
4. 历史沿革补充材料清单（逐项列明材料、核查目的、对应股权变动事项）。

交付前复核：
- 股权变动按时间顺序完整覆盖，无遗漏
- 每个事实均可对应到底稿页码或拆分PDF
- 拆分PDF可打开、可读、无漏页、无误纳入无关材料
- 正文排除所有非股权事项
- 所有`【待核验】`事项均已纳入补充材料清单

### （三）第三章 公司的股东和实际控制人
1. 严格参照模板该章节的结构、行文方式、表格形式与信息颗粒度起草。
2. 内容覆盖：
   （1）公司全体股东及持股情况；
   （2）逐一列示每名股东基本情况；股东为公司/合伙企业的，继续穿透列示其股东/合伙人及对应持股、出资比例；
   （3）股权质押、冻结情况；
   （4）控股股东及实际控制人认定；
   （5）一致行动关系；
   （6）员工持股平台及股权激励情况。
3. 删除模板中与本项目不适用的旧内容；资料无法确认处用`【】`占位，不得编造。

### （四）第四章 公司的附属公司
1. 沿用模板的结构、表格、编号和行文风格。
2. 逐一列表说明各附属公司的：企业名称、统一社会信用代码、登记状态、法定代表人、注册资本、住所、成立日期、营业期限、经营范围、主要人员、股权结构。
3. 分类列示：境内存续附属公司、境外附属公司、已注销历史附属公司；说明直接/间接持股关系；已注销主体仍需列示基本情况与注销状态。
4. 资料无法确认的内容用`【】`占位，不得推测编造。

### （五）第五章 公司的业务
1. 依据目标公司及子公司企查查报告，按照模板第五章写法起草。
2. 内容包含：经营范围、主营业务、经营资质、问题与分析。
3. 保留模板原有格式与结构，不得随意调整层级。

### （六）附件一 公司的关联方情况
1. 按照模板附件一的结构与格式起草，页面方向为纵向。
2. 关联方分类列示：
   1. 关联自然人：持股5%以上自然人、董监高、控制公司的法人的董监高、关系密切家庭成员；
   2. 关联法人：控制公司的法人、持股5%以上法人、关联自然人控制或任职的企业、过去12个月内关联方、子公司。
3. 同一企业存在多种关联关系的，需全部列明。
4. 不撰写关联交易部分内容；同业竞争内容放在正文第六章。
5. 无法确定的内容用`【】`占位。

### （七）第七章 公司的主要财产（含附件二）
1. 沿用模板结构、表格与写作口径，仅修改第七章及附件二。
2. 重点核查与撰写：土地房产、租赁物业、主要设备、知识产权。
3. 附件二同步更新要求：
   - 商标列示商标图案；
   - 专利仅列已授权专利，共有专利完整列明全部权利人；
   - 软件著作权按登记信息列表；
   - 公司及现存附属公司的全部知识产权纳入并去重。
4. 权利受限、资料缺失或无法确认的内容用`【】`标注，不作推定。

### （八）第九章 公司治理
1. 依据公司章程、工商资料、董监高调查资料、任职报告及三会文件起草。
2. 内容覆盖：公司组织结构；现任董事、监事及高级管理人员列表；报告期内人员变动；主要兼职情况；治理制度及股东会、董事会、监事会运行情况；存在的法律问题及整改建议。
3. 无法确认的任期、选任程序、兼职或制度运行情况用`【】`标注。
4. 同步检查手工编号与自动编号，不得出现重复。

## 三、后续章节扩展格式
后续新增其他尽调章节或工作模块时，统一沿用以下结构：
